# DevFlow Knowledge Base

> Accumulated learnings across development cycles. Referenced by Brainstormer and Architect at cycle start. Updated by Finalizer at cycle end. Versioned in git — survives across conversations.

---

## How to Use

- **Brainstormer (Phase 1):** Read this file before asking questions. Check if a similar feature was built before — reuse the patterns.
- **Architect (Phase 3):** Read this file during exploration. Avoid re-exploring patterns already documented.
- **Planner (Phase 4):** Read this file during pattern exploration (Step 6). Reuse planning patterns and avoid known pitfalls.
- **Implementer (Phase 5):** Read this file at Step 1. Apply documented implementation patterns and avoid known mistakes for the detected stack.
- **Reviewer (Phase 6):** Read this file at Step 1 (both Cycle and Standalone mode). Flag documented anti-patterns as findings when matched.
- **Debugger (Phase 7):** Read this file at Step 1. Check if a documented anti-pattern or known root cause explains the failure.
- **Finalizer (Phase 8):** Append new learnings after each cycle completes.

## Format

Each cycle adds a section with: patterns discovered, anti-patterns to avoid, and key architectural decisions.
