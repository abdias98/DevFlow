# DevFlow Knowledge Base

> Accumulated learnings across development cycles. Referenced by all mid-cycle agents. Updated by Finalizer at cycle end. Versioned in git — survives across conversations.

---

## How to Use

- **All agents:** Read the **By Topic** section first — find your topic, read only that section. If your topic isn't there, fall back to **Cycle History** and scan for relevant entries.
- **Brainstormer (Phase 1):** Read before asking questions. Check if a similar feature was built before — reuse the patterns.
- **Architect (Phase 3):** Read during exploration. Avoid re-exploring patterns already documented.
- **Planner (Phase 4):** Read during pattern exploration (Step 6). Reuse planning patterns and avoid known pitfalls.
- **Implementer (Phase 5):** Read at Step 1. Apply documented implementation patterns and avoid known mistakes for the detected stack.
- **Reviewer (Phase 6):** Read at Step 1 (both Cycle and Standalone mode). Flag documented anti-patterns as findings when matched.
- **Debugger (Phase 7):** Read at Step 1. Check if a documented anti-pattern or known root cause explains the failure.
- **Finalizer (Phase 8):** Append new learnings after each cycle completes. Add to **By Topic** (preferred) and **Cycle History** (chronological log).

## Format

Two sections:

1. **By Topic** — patterns and anti-patterns grouped by topic (testing, security, architecture, performance, stack-specific). This is the primary section agents should read — it allows loading only the relevant topic instead of the entire file.
2. **Cycle History** — chronological log of learnings per cycle. Used for traceability and as a fallback when a topic isn't in **By Topic** yet.
